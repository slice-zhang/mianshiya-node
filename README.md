# 面试鸭后端服务 (Mianshiya-Node)

基于 Express 5 搭建的企业级面试题库管理系统后端服务

## 项目概述

**面试鸭**是一个专注于面试题库管理的全栈应用，本项目是其后端服务部分。系统提供了完整的用户管理、题库管理、题目管理、标签管理、文件上传等核心功能，支持多环境部署和完善的权限控制机制。

### 核心功能
- **用户系统**：注册、登录、权限管理、分页查询、个人资料管理
- **题库管理**：创建、更新、删除、分页查询题库，支持成人内容审核
- **题目管理**：题目增删改查、关联题库、成人内容审核日志
- **标签管理**：标签的CRUD操作，便于题目分类
- **文件服务**：集成阿里云OSS，支持图片等资源上传
- **安全机制**：基于Redis的Session管理、IP黑名单、请求参数限制
- **日志系统**：完整的访问日志和错误日志记录

## 技术架构

### 整体架构
采用经典的 **MVC 分层架构** + **插件化设计**，实现了高内聚低耦合的代码组织方式：

```
┌─────────────────┐
│    Express 5    │ ← HTTP Server
└────────┬────────┘
         │
┌────────▼────────┐
│   Router层      │ ← 路由分发 + 中间件处理
└────────┬────────┘
         │
┌────────▼────────┐
│   Service层     │ ← 业务逻辑处理
└────────┬────────┘
         │
┌────────▼────────┐
│    Model层      │ ← 数据模型 + Sequelize ORM
└────────┬────────┘
         │
┌────────▼────────┐
│   Plugin层      │ ← 数据库、Redis、OSS、Logger等插件
└─────────────────┘
```

### 核心技术栈

| 类别 | 技术 | 版本 | 用途 |
|------|------|------|------|
| **Web框架** | Express | 5.2.1 | HTTP服务基础框架 |
| **数据库** | MySQL + Sequelize | 6.37.7 | 关系型数据存储与ORM |
| **缓存** | Redis | 5.10.0 | Session存储与缓存 |
| **对象存储** | 阿里云OSS | 6.23.0 | 文件上传与存储 |
| **安全** | bcryptjs | 3.0.3 | 密码加密 |
| **验证** | Joi | 18.0.2 | DTO参数校验 |
| **工具库** | lodash-es, dayjs, shortid | - | 工具函数、时间处理、短ID生成 |
| **文件处理** | multer + sharp | 2.0.2 + 0.34.5 | 文件上传与图像处理 |
| **开发工具** | nodemon, cross-env | - | 热重载、跨平台环境变量 |

### 架构特色

1. **模块化设计**：通过 `module-alias` 实现 `@` 别名，简化模块引用路径
2. **插件化架构**：数据库、缓存、存储、日志等服务以插件形式注入，便于维护和扩展
3. **中间件机制**：身份验证、管理员权限、文件处理等通过中间件实现，职责清晰
4. **优雅关闭**：支持 SIGINT/SIGTERM/SIGQUIT 信号，确保资源正确释放
5. **多环境配置**：支持 development/test/production 三种环境配置
6. **错误处理**：统一的错误处理和日志记录机制
7. **安全防护**：IP黑名单、请求参数限制、Session安全配置

## 快速开始

### 环境要求
- Node.js (LTS 版本)
- PNPM (10.17.1)
- MySQL 8.0+
- Redis 服务器
- 阿里云 OSS 账号（可选，用于文件上传）

### 安装与运行
```bash
# 安装依赖
pnpm install

# 开发环境启动
pnpm run start:dev

# 测试环境启动  
pnpm run start:test

# 生产环境启动
pnpm run start
```

### API 文档
所有 API 接口均遵循 RESTful 设计原则，返回统一的 JSON 响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
